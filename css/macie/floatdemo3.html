Problems with floating images

Here are four floating images that should all be links to the WSP Homepage. MacIE will only display the image floated in some situations. Here, the floating is done by setting the float property on the containing p ("0") or div ("1") element or on the anchor ("2").

MacIE does correctly float the image in some cases, for example, when the float property is set on the image itself ("3"). This erratic floating can cause serious problems even with pages that degrade gracefully if a feature (in this case, the float property) is not implemented.

Even more importantly, when it is a p element that is floated ("0"), the image does not act like a link. This loss of the link information is a serious obstacle to the use of this otherwise safe technique for placing content outside of the normal flow.
