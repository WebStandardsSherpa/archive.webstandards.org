Inheritance breakdown within tables

Introduction

One bug present in IE4 is that style inheritance (whereby container objects pass their styles on to their children) breaks down when entering a table element.

Example

This is a table within a DIV container. The DIV has had the following styles applied to it:

DIV#d1 { font: 900 italic 0.8em "Verdana", "Arial", "Helvetica", "Helv", sans-serif;
         color: green; background: white none;

The table has had no styles explicitly given, yet its contents should have exactly the same style as the text in the DIV. (Note - to ensure that this bug is visible, the rest of the document has been explicitly given a different value for each of the above properties.)
