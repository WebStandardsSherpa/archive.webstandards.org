WaSP標準提案

摘要
 

萬維網聯盟(World Wide Web Consortium,簡稱W3C) 早已定立標準, 解讀網上的內容.

瀏覽器製家出品不全面支
援這些標準的瀏覽器, 正正傷害著網絡開發者, 商家及用家等人.

當W3C標準失去一致的支援時,會令網絡技
術的使用和發展, 構成不必要的阻礙和使費.

我們理解在日新月異的市場埵陶郱s的需要. 不過, 這樣的創新, 動搖了存在多時
的 W3C 標準的支援基礎, 不必要地分裂網絡, 為禍人間.

我們的目的正是要支持這些標準的核心, 並鼓勵瀏覽器製家沿用之, 如是為大家確立簡單, 人人可用的網絡技術.

細節
 

當我們說網絡"標準"時, 係指:

結構語言
HTML 4.0
XML 1.0

格式語言
Cascading Style Sheets 1
Cascading Style Sheets 1
XLS (建立中)

物件模型
Document Object Model (DOM) 1 Core HTML/XML

程式語言
ECMAScript ( JavaScript "官方"版)

. ... 以及新出的標準, 譬如那些用於電視和個人數碼助理(PDA)的瀏覽器

這些創自 W3C 的標準 (ECMAScript 除外), 旨在對開發者於技
術的追求 (先進的畫面顯示和互動特色)與網絡取
讀性(瀏覽器(和其他客戶裝置)的容納極限及環境)之間取個平衡.

每層網上的文件如上設計便
可達到平衡. 這就是何以結構性的 HTML 或 XML, 從一件文件表示中拆開是如斯重要, 何以擁有一般性及可預測的物件模型是個關鍵. 而且這亦是何以這些被我們全力支持的核心標準, 定是瀏覽器製家於試圖增加其產品獨有的創新時必須照顧.

HTML, XML, CSS 和 DOM 何只一組有趣的枝術. 他們實是一舉兩得(先進的畫面顯示與廣泛的取讀性)的網頁創作方法.

如是者可令開發者從心所欲地工作. 可是恰值目前四分五裂, 經常互不兼容的情況下, 這樣做非常艱難. 由於瀏覽器間的互不兼容, 故同一網頁須製多個版本, 這對開發者及其客戶而言都是浪費及自敗的行為. 另一個常見的做法是設法逃避,此舉徒然製造負擔 - 阻礙新出的電視和個人數碼助理靈活地取讀網頁.

是故, 這些技術, 不只須開發者支持, 而且有賴瀏覽器製家的支援承諾.
